# AGENTS.md

## 项目协作约束

本文件用于约束本仓库内的代理协作行为。

## 依赖管理

- 凡是项目级依赖安装操作，必须先告知用户，不能直接执行。
- 凡是项目级依赖升级、删除、替换操作，必须先告知用户，不能直接执行。
- 凡是会修改 `package.json`、`pnpm-lock.yaml` 的依赖管理操作，默认都视为需要先告知用户。
- 包括但不限于以下命令：
  - `pnpm install`
  - `pnpm add`
  - `pnpm remove`
  - `pnpm update`
  - `npm install`
  - `npm add`
  - `npm remove`
  - `npm update`

## 当前项目约定

- 当前项目使用 `TypeScript`
- 包管理器优先使用 `pnpm`
- 在未得到用户明确确认前，不主动执行任何项目级依赖变更

## 执行原则

- 优先修改代码、配置和文档
- 涉及依赖变更时，先说明原因、影响和将执行的命令，再等待用户确认

## 回复规则

- 当回复中涉及当前项目代码文件路径时，不要从磁盘盘符或仓库绝对路径开始；统一从 `/apps` 开始书写到具体文件位置，例如 `/apps/server/src/index.ts`，避免路径过长影响阅读。
